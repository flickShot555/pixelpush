1. Introduction
PixelPush is a web-based Progressive Web App (PWA) designed for developers who want to
transform their GitHub contribution graph into pixel art. By connecting their GitHub account,
users can select a visual theme, generate a pixel art image mapped to the graph's green
shading system, and receive a precise commit schedule that guides them to recreate that image
on their live GitHub profile — one contribution at a time.
This document covers the complete product specification, technical architecture, feature set,
API integrations, and development roadmap for PixelPush, built and maintained by
Aepostrophee.
2. Product Overview
2.1 Vision
Every developer has a GitHub contribution graph. Most ignore it. PixelPush turns it into a
canvas — making the act of writing code a creative, gamified, and socially shareable
experience.
2.2 Core Value Proposition
• Transforms a developer's existing GitHub profile into interactive pixel art
• Removes the complexity of manually planning contribution patterns
• Provides a guided, day-by-day commit schedule to complete artwork
• Creates shareable progress milestones that market the app organically
2.3 Target Audience
Segment Description
Primary Software developers active on GitHub with consistent contribution
history
Secondary Open source contributors seeking creative profile differentiation
Tertiary Developer communities, bootcamp students, and tech content
creators

3. Feature Specification
3.1 GitHub OAuth Integration
Users connect their GitHub account via OAuth 2.0. The app requests the minimum required
scopes to read contribution data and guide the user through future commits.
OAuth Scope Purpose
read:user Access basic profile information (username, avatar)
repo Required to read contribution data across public and private repos
user:email For account identification and notification delivery
3.2 Contribution Graph Reading
Once authenticated, PixelPush queries GitHub's GraphQL API to retrieve the user's full
contribution calendar. The API returns contribution data for each day of the past year including:
• Date of contribution
• Exact contribution count
• Hex color code (one of 5 shade levels used by GitHub)
The five shade levels correspond to:
Hex Color Shade Level
#ebedf0 Level 0 — No contributions (empty)
#9be9a8 Level 1 — Low contributions (light green)
#40c463 Level 2 — Moderate contributions (mid green)
#30a14e Level 3 — High contributions (dark green)
#216e39 Level 4 — Very high contributions (darkest green)
Important: GitHub's shading thresholds are relative to each user's personal contribution history,
not fixed numbers. PixelPush calculates a per-user threshold baseline from historical data to
generate accurate future commit targets.
3.3 Theme Selection & Pixel Art Generation
Users select a visual theme for their pixel art. PixelPush's generation engine maps theme
imagery to the 5 available shade levels of the GitHub graph, producing a viable pixel art design
within the 52 columns × 7 rows resolution of the contribution grid.

For each theme, the app generates 3 to 5 pixel art variations. Each is previewed as a grid
overlay matching the exact dimensions of the user's GitHub graph. The user selects one design
to proceed with.
3.4 Commit Schedule Generator
After a design is selected, PixelPush calculates a forward-looking commit schedule. This
schedule tells the user:
• Which days require contributions
• How many commits to make on each active day
• At what intervals to space commits for natural-looking graph shading
The schedule is displayed as a calendar view with daily tasks. Users can export the schedule or
receive it as daily reminders via browser push notifications or email.
Schedule Recovery
If a user misses a day, PixelPush offers a 'Recalculate from Today' feature that regenerates the
remaining schedule dynamically, adjusting future commit targets to compensate for missed days
without losing the overall design.
3.5 Progress Tracker
PixelPush continuously monitors the user's GitHub contribution graph and compares it against
the target design. The progress tracker displays:
• Overall completion percentage
• A live side-by-side view of target design vs. current graph
• Upcoming commit tasks for the next 7 days
• Streak tracking — consecutive days on schedule

4. Technical Architecture
4.1 Technology Stack
Layer Technology

Styling Tailwind CSS
Authentication NextAuth.js with GitHub OAuth 2.0 provider
GitHub Data GitHub GraphQL API v4
Pixel Art Rendering HTML5 Canvas API / D3.js
Image Card Generation Satori + resvg-js (server-side OG image rendering)
GIF Generation gifencoder (Node.js, server-side)
Database PostgreSQL via Prisma ORM
Push Notifications Web Push API (VAPID)
Hosting & Deployment Vercel (frontend + API routes)
CDN & Storage Vercel Blob Storage for generated image cards
4.2 System Architecture Overview
PixelPush follows a serverless architecture pattern, with Next.js API routes handling all backend
logic. This eliminates the need for a separate backend server and keeps the infrastructure
simple and cost-effective during the early growth phase.
Core Data Flow
1. User lands on PixelPush and clicks 'Connect GitHub'
2. GitHub OAuth flow initiates — user authorizes the app
3. NextAuth.js stores the access token securely in a server-side session
4. PixelPush calls GitHub GraphQL API to fetch the full contribution calendar
5. The contribution grid (52 × 7 days) is parsed, normalized, and stored
6. User selects a theme — the pixel art engine generates design candidates
7. User picks a design — the schedule generator produces a daily commit plan
8. Progress tracker runs on a cron schedule (daily) to update completion status
9. Share cards are generated on demand and cached in Vercel Blob Storage

4.4 Pixel Art Engine
The pixel art generation engine converts theme images into valid GitHub graph art through the
following pipeline:
10. Source image is downsampled to 52 × 7 pixels (matching the contribution grid)
11. Each pixel is quantized to one of 5 brightness levels
12. Brightness levels are mapped to GitHub's 5 shade hex values
13. Multiple design variations are generated using different cropping and style parameters
14. Each variation is rendered as an interactive grid preview using the HTML5 Canvas API
4.5 Share Card Generation
Share image cards are generated server-side using Satori (JSX to SVG) combined with resvg-js
(SVG to PNG). Cards are generated on demand and cached to Vercel Blob Storage with a
unique URL.
The share card API endpoint accepts:
• username — GitHub username
• stage — started / progress / completed
• percent — completion percentage (for progress cards)
Generated cards are returned as PNG images optimized for Twitter (1200 × 630px) and are
cached for 24 hours.

7. Notifications & Reminders
7.1 Push Notifications
PixelPush implements the Web Push API using VAPID keys for browser-based push
notifications. Notifications are sent via a daily cron job that runs at 8:00 AM in the user's local
timezone.
Notification Type Trigger
Daily Reminder Sent on days the user has a scheduled commit target
Streak Alert Sent when a user is at risk of breaking a streak (commits due by
midnight)
Milestone Sent when user crosses 25%, 50%, 75%, and 100% completion
Completion Sent when the full design is finished — includes share prompt
7.2 Email Notifications
As a fallback for users who decline push notifications, PixelPush sends daily schedule digests
and milestone alerts via email. Emails are delivered through a transactional email provider
(Resend or Postmark).

9.4 AI Project Suggestion Engine
The project suggestion feature is the most technically distinctive Pro capability. It uses the
GitHub API data already collected during onboarding — specifically the user's public
repositories — to infer their primary programming languages, frameworks, and project patterns.
This data is combined with the selected design theme and schedule duration to generate highly
personalized project recommendations via the Claude API.
How It Works
37. On design activation, PixelPush reads the user's public repos via GitHub API
38. Languages, frameworks, and project types are extracted and ranked by frequency
39. The design theme (e.g. Aviation) and schedule duration (e.g. 4 months) are noted
40. A structured prompt is sent to the Claude API with this context
41. Claude returns 3 to 5 project suggestions with titles, descriptions, estimated commit
frequency, tech stack, and portfolio value rating
42. Suggestions are displayed on the Pro dashboard alongside the commit schedule
Example Output
User: Python developer · Theme: Aviation · Schedule: 4 months
Suggested Project: FlightPathCLI — A Python CLI tool that parses ADS-B flight data and visualises
real-time aircraft trajectories using matplotlib. Estimated commits: 3-4 per week. Portfolio Value: High
— demonstrates API integration, data parsing, and CLI design.
Why This Feature Justifies Pro
• Solves the ethical tension — users make real, meaningful commits instead of empty
ones
• Adds career value independent of the graph art — users finish with a portfolio piece
• Highly personalized — no two users get the same suggestions

10. Security Considerations
10.1 OAuth Token Handling
• GitHub OAuth access tokens are encrypted at rest using AES-256
• Tokens are stored server-side only and never exposed to the client
• NextAuth.js handles session management with HTTP-only cookies
10.2 Rate Limiting
• GitHub GraphQL API allows 5,000 points per hour per authenticated user
• PixelPush batches API calls and caches results to stay well within limits
• Contribution data is synced once every 6 hours to minimize API usage
10.3 Data Privacy
• PixelPush only reads contribution metadata (dates, counts, colors) — never code or
repository content
• Users may revoke GitHub access at any time from GitHub's application settings
• User data is permanently deleted within 30 days of account deletion

11. Glossary
Term Definition
Contribution Graph The 52-column x 7-row heatmap on a GitHub profile showing
commit activity over the past year
Shade Level One of 5 green intensity values GitHub uses to represent
contribution density on any given day
Pixel Art Engine PixelPush's internal system for converting theme images into 52x7
grid designs mapped to shade levels
Commit Schedule The day-by-day plan generated by PixelPush specifying how many
GitHub contributions to make on each date
Share Card A dynamically generated PNG image (1200x630px) used for social
media sharing of progress milestones
PWA Progressive Web App — a web application installable on mobile and
desktop with native-like capabilities
VAPID Voluntary Application Server Identification — a protocol used to
send Web Push notifications securely
Satori A library developed by Vercel for converting JSX markup to SVG,
used for server-side share card generation
AI Project Suggestion A Pro feature that generates personalized GitHub project ideas
matched to the user's tech stack and design theme using the
Claude API
Portfolio Fit Score A rating assigned to each AI-generated project suggestion indicating
its value for software engineering interviews and portfolios
MRR Monthly Recurring Revenue — total monthly income from active Pro
subscriptions
Lifetime Deal A one-time payment option offered at launch giving permanent Pro
access, capped at a fixed number of redemptions
Serverless Function A backend code execution model used by Vercel where functions
run on-demand per request with no always-on server required
